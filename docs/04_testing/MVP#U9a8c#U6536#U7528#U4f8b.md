# MVP 验收用例

| 编号 | 场景 | 预期 |
|---|---|---|
| MVP-01 | 打开工作台 | 能看到当前项目、任务、风险和流程状态 |
| MVP-02 | 查看资料中心 | 能识别资料版本、解析状态和缺失项 |
| MVP-03 | 查看事实与指标 | 每个事实能看到状态、来源和责任人 |
| MVP-04 | 打开编制工作台 | 能按章节查看编制状态、引用和审核状态 |
| MVP-05 | 查看质量问题 | 阻断和严重问题被突出展示 |
| MVP-06 | 进入成果中心 | 能看到 Word、Excel、归档包等成果状态 |
| MVP-07 | 调用 API | `/health` 和 `/api/bootstrap` 返回正常 |
| MVP-08 | 新建项目 | 项目写入 PostgreSQL，并在项目中心刷新可见 |
| MVP-09 | 登记资料 | 资料写入 PostgreSQL，初始状态为待解析 |
| MVP-10 | 发起资料解析 | 解析任务写入 `parse_jobs` 并产生审计记录 |
| MVP-11 | 确认和锁定事实 | 事实状态更新并可刷新回显 |
| MVP-12 | 提交和审核章节 | 章节状态可从编制中流转到待审核、已审核 |
| MVP-13 | 关闭质量问题 | 质量问题状态更新为已关闭 |
| MVP-14 | 请求成果导出 | 存在未关闭阻断问题时成果状态保持受阻 |
| MVP-15 | 上传资料文件 | 文件写入本地存储，资料元数据写入 PostgreSQL |
| MVP-16 | 执行资料解析 | 解析任务完成，资料状态更新为已解析 |
| MVP-17 | 生成成果文件 | 阻断问题关闭后成果状态为已生成，并可下载文件 |
| MVP-18 | 生成章节初稿 | 章节内容由已确认事实生成，章节状态更新为编制中 |
| MVP-19 | 查看模板知识库 | 前端知识页可展示 PG 中的模板和知识条目 |
| MVP-20 | 查看系统审计 | 系统页可展示用户角色和最近审计记录 |
| MVP-21 | 运行质量检查 | 质量检查任务完成，并按规则追加问题记录 |
| MVP-22 | 登录认证 | 使用演示账号登录后返回 token 和当前用户 |
| MVP-23 | 写接口权限 | 未登录调用章节生成等写接口返回 401 |
| MVP-24 | 引用回链 | 章节生成后自动写入引用摘录和来源 |
| MVP-25 | 质量规则展示 | 系统页可展示启用的质量规则 |
