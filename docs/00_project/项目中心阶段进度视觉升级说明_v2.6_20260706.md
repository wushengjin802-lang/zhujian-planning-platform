# 项目中心阶段进度视觉升级说明（v2.6）

## 本次范围

MVP开发2：将项目中心及项目七阶段相关页面顶部的阶段进度条，按照用户提供的参考图进行视觉升级，并作为后续版本的统一阶段导航基准。

## 已完成内容

1. 新增 `apps/web-vue/src/components/PhaseProgress.vue` 统一阶段进度组件。
2. 在 `apps/web-vue/src/App.vue` 中替换原有 `workflow-strip`，使项目中心、资料中心、事实与指标、编制工作台、GIS与投资测算、质量审查、成果中心等七阶段页面统一使用新样式。
3. 阶段高亮逻辑改为：优先根据当前路由匹配阶段；非阶段页面根据当前项目 `phase` 回退显示当前阶段。
4. 项目中心页面移除旧版重复阶段进度块，避免顶部出现两套阶段导航。
5. 视觉上实现白色大圆角卡片、圆形编号、箭头连接、当前阶段绿色高亮、发光连接线和当前阶段底部短横线。
6. 增加响应式处理，小屏下允许横向滚动，保证七阶段名称不挤压错位。
7. 根 `package.json`、前端 `apps/web-vue/package.json`、`package-lock.json` 版本号统一升级为 `2.6.0`。
8. 在 `docs/00_project` 下新增完成度 Checklist：`住建平台功能完成度Checklist_v2.6_20260706.xlsx`。

## 验证情况

- `npm run typecheck`：已通过。
- `npm run build`：当前 Linux 沙箱中的上传包只包含 Windows 平台 Rollup 可选依赖，缺少 `@rollup/rollup-linux-x64-gnu`，因此 Vite build 未能在沙箱中完成。该问题属于依赖安装环境问题，不是本次代码语法问题；在目标环境解压后执行 `npm install`，再执行 `npm run build` 即可拉取当前平台对应依赖。
