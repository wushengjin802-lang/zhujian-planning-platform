# 工作台边界能力完善说明（2026-07-06）

本次在上一版“工作台增强”的基础上，继续补齐尚未完成的工作台边界能力，目标是让工作台从“实时聚合展示页”进一步升级为“可处理、可审核、可通知、可追踪任务”的第一阶段作业入口。

## 一、完成范围

### 1. 持久化工作项 `work_items`

新增工作项模型与迁移，支持：

- 由资料、事实、章节、质量问题等业务对象自动物化工作项；
- 工作项优先级 P0-P3；
- 工作项状态：待领取、待处理、处理中、已完成、已取消；
- 领取、完成、转交接口；
- 负责人、处理人、来源对象、业务路由、完成时间和审计记录。

新增接口：

```text
POST /api/work-items/{item_id}/claim
POST /api/work-items/{item_id}/complete
POST /api/work-items/{item_id}/transfer
```

### 2. 持久化审核任务 `review_tasks`

新增审核任务模型与迁移，支持：

- 章节待审核自动形成审核任务；
- 投资估算计算完成后自动形成确认任务；
- 审核通过和审核退回；
- 审核人、提交人、审核意见、决策时间和审计记录；
- 审核通过后联动章节状态或投资估算确认状态；
- 审核退回后联动章节回到“编制中”或估算回到“draft”。

新增接口：

```text
POST /api/review-tasks/{task_id}/approve
POST /api/review-tasks/{task_id}/reject
```

### 3. 持久化通知 `notifications`

新增通知模型与迁移，支持：

- 严重/阻断质量问题自动生成未读通知；
- 审核退回自动生成提醒；
- 通知状态：未读、已读、已归档；
- 通知已读接口；
- 通知与项目、来源对象、路由和级别关联。

新增接口：

```text
POST /api/notifications/{notification_id}/read
```

### 4. 异步任务取消、重试和卡住检测

工作台任务中心新增：

- `taskKind`：parse / quality / artifact；
- `rawStatus`、`canCancel`、`canRetry`、`stuck` 字段；
- 运行/排队超过阈值自动标记“疑似卡住”；
- 解析任务取消后资料进入“需复核”；
- 质量检查任务可取消；
- 成果导出任务取消后标记“受阻”；
- 失败、取消或卡住任务可重试。

新增接口：

```text
POST /api/tasks/{task_id}/cancel
POST /api/tasks/{task_id}/retry
```

### 5. 项目成员级可见性过滤

新增 `ProjectMember` ORM 模型，并在工作台聚合中启用项目可见性过滤：

- 管理类角色：管理员、负责人、领导、审核、经理，可查看全部项目；
- 普通角色：仅查看自己作为项目成员或项目负责人的项目；
- 工作项、审核任务和通知按可见项目过滤；
- 操作接口执行项目可见性校验。

### 6. 前端工作台交互

工作台页面新增：

- 待办事项行内“领取 / 完成”；
- 待审核任务行内“通过 / 退回”；
- 通知“已读”；
- 任务中心“取消 / 重试 / 查看”；
- 卡住任务以“疑似卡住”高亮展示；
- 工作项显示处理人，未分配时显示业务负责人。

## 五、仍未完全关闭的边界

本次已将工作台边界从“派生展示”推进到“持久化处理”。仍建议后续继续完善：

- 工作项到期时间、逾期提醒和批量处理；
- 工作项处理历史和评论流；
- 审核会签、转交、指定审核人规则；
- 通知订阅策略、用户定向推送和站内消息中心；
- Celery 原生 revoke 与 Worker 安全停止点；
- 更细粒度任务日志、错误码和阶段进度；
- 全量业务接口的 RBAC 和项目级权限过滤。
