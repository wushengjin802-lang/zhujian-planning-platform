# 项目中心阶段进度单行布局修复说明 v2.7

## 一、问题说明

v2.6 版本中，项目中心上部阶段进度在部分宽度下出现两行展示，与参考图的“一行贯穿式阶段进度”差距较大。

根因是 `PhaseProgress.vue` 中步骤节点和连接箭头作为同级 grid 元素渲染，实际元素数量为 7 个阶段 + 6 个连接箭头 = 13 个，但样式只定义了 `repeat(7, ...)`，因此浏览器会自动换行。

## 二、本次修复

1. 将阶段进度轨道从 `grid` 改为 `flex`。
2. 设置 `flex-wrap: nowrap`，强制 7 个阶段和 6 个箭头保持同一行。
3. 保留阶段标题、副标题 `white-space: nowrap`，避免文字折行。
4. 在窄屏场景下使用横向滚动兜底，不再换成两行。
5. 保留 v2.6 已有视觉风格：当前阶段绿色高亮、箭头发光、当前阶段下方短线。
6. 版本号升级为 `2.7.0`。

## 三、涉及文件

- `apps/web-vue/src/components/PhaseProgress.vue`
- `package.json`
- `apps/web-vue/package.json`
- `package-lock.json`
- `docs/00_project/住建平台功能完成度Checklist_v2.7_20260706.xlsx`

## 四、验证结论

- 阶段进度不再换行。
- 项目建档、资料清点、事实确认、章节编制、分析测算、专业复核、成果输出七个阶段保持同一横排。
- 当前阶段根据路由和当前项目阶段继续自动高亮。
- 本次不修改业务数据结构和后端接口。
