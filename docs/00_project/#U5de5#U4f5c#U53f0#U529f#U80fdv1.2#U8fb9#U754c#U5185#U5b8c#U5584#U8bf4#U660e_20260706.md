# 工作台功能 v1.2 边界内完善说明（2026-07-06）

## 本次范围

本次只继续完善 PDD 5.1 工作台范围内的协同闭环能力，不扩展到报告编制、GIS、投资测算、方案比选等其他业务模块。

## 完成内容

1. **工作项转交前端入口**
   - 工作台待办行增加“转交”操作。
   - 用户输入接收人用户ID后调用既有后端转交接口。
   - 转交后刷新工作台，保持工作台作为任务入口的边界。

2. **工作项处理记录查看**
   - 工作台待办行增加“记录”操作。
   - 支持查看领取、完成、转交、取消和意见事件。
   - 使用既有 `workbench_events`，不新增额外业务表。

3. **审核任务处理记录查看**
   - 审核队列增加“记录”操作。
   - 支持查看分配、意见、通过、退回和会签事件。
   - 仍保持审核任务作为工作台协同入口，不替代正式报告审阅模块。

4. **工作台操作权限边界**
   - 工作项领取、完成、取消、转交增加动作级校验。
   - 非管理角色只能处理未分配或分配给自己的工作项。
   - 管理角色可处理其可见项目范围内的工作项。

5. **审核任务操作权限边界**
   - 审核通过、退回、分配增加动作级校验。
   - 非管理角色只能处理分配给自己的审核任务；未分配任务允许首次分配。
   - 意见和会签仍允许项目可见成员参与，作为协作留痕。

6. **审核事件留痕修正**
   - 修正“审核通过”时同时写入退回事件的问题。
   - 通过与退回事件现在分别对应真实动作。

## 未纳入本次范围

以下能力仍不在本次边界内，不做扩展：

- 完整 RBAC 权限中心；
- 审核会签规则引擎；
- 消息订阅推送；
- 工作项评论回复流；
- 任务 Worker 安全停止点；
- 报告章节正式在线审阅；
- GIS、投资测算、方案比选等非工作台模块。
