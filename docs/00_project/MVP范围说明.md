# MVP 范围说明

## 建设目标

第一阶段交付一套可运行、可验证、可继续扩展的最小业务闭环，支撑可研报告等典型成果从资料归集到归档输出。

## 纳入范围

- 项目建档、项目状态、成员与角色视图；
- 资料清单、文件版本、解析状态和缺失项提示；
- 事实与指标库，支持来源追溯、人工确认、锁定和修订；
- 报告章节工作台，支持章节状态、引用、AI 建议占位和审核流转；
- 基础知识与模板库；
- 基础 GIS/投资测算入口和复算记录占位；
- 质量门禁，覆盖缺失项、事实冲突、数值异常和引用缺失；
- 成果中心，覆盖 Word、Excel、决策记录和归档包状态；
- 基础系统配置、审计和模型网关占位。

## 暂缓范围

- 完整 BIM/CAD 专业设计能力；
- 自动生成施工图或高精度效果图；
- 替代结构、设备、节能等专业计算；
- AI 自动审批、冻结或发布成果；
- 第三阶段的完整方案决策与分层冻结闭环。

## MVP 退出标准

- 能完整演示“上传资料、解析、事实确认、章节生成、审核、输出、归档”；
- 关键事实和引用可追溯到原始资料；
- 严重质量问题能够阻止成果发布；
- 至少一个历史项目样本可完整复现；
- 新增任务包不需要重构核心架构。

## 当前实现状态

已完成工程骨架、PostgreSQL 接入、PostGIS/pgvector 扩展启用、项目/资料/事实/章节/质量/成果主链路 CRUD、资料上传、本地/MinIO 对象存储适配、资料解析状态推进、可执行 Celery 任务与同步兜底、章节初稿生成、质量检查任务、成果生成下载、用户角色展示、模板知识库展示、审计日志查询、登录会话、写接口权限校验、质量规则配置展示、章节引用回链、Word/Excel/PPT/归档包 Python 工具链导出、验收样本和端到端验收脚本。

仍需在验收前加强：

- Redis、MinIO、LibreOffice 在目标部署环境中的服务级联调；
- 真实历史项目资料脱敏导入，并由业务人员确认抽取结果；
- OCR/VLM 对扫描 PDF 和图片资料的增强解析；
- 投资估算、GIS 区位分析从 MVP 入口升级为可复算业务模型；
- 正式报告模板的版式、页眉页脚、编号和企业标准样式精修。
